1. Fact : 무슨 일이 일어났지?#

DataBase란!#

먼저 흔히 말하는 DB에 대해 비전공자도 이해하기 쉽게 Excel에 비유해서 설명해주었다. DB란 결국 우리가 저장하고 관리하는 데이터, 즉 정보들이 정리되어 있는 그릇이라는 것이다. 서점을 대상으로 고객, 운영자, 관리자의 요구사항 명세서를 바탕으로 DB를 설계해보는 실습을 진행했다.

Django 기반 DB 구축해보기!#

Django 프레임워크에서는 DB를 models.py 파일에서 관리한다. 코드를 보고 따라 치는 방식이었지만, 그 과정에서 Entity를 구현하는 방법과 구조에 대해 자연스럽게 익힐 수 있었다. 추가로 admin 페이지 세팅과 터미널을 이용한 CRUD 실습도 함께 진행했다.

Comment Entity 생성 및 Create!#

User 엔티티를 코드를 따라 치며 만들어보는 실습 이후, 주어진 조건에 맞는 Comment(댓글) 테이블을 생성해보고 Admin 페이지를 통해 댓글을 직접 추가해보는 실습을 진행했다.

2. Feeling : 무엇을 느꼈지?#

발전?#

이전 과제를 진행하면서 데이터를 계속 list나 dictionary 구조로만 설계하다 보니, 서버를 닫거나 프로그램을 종료하면 데이터가 항상 리셋된다는 점이 아쉬웠다. 이번에 DB를 통해 데이터를 영구적으로 저장해보면서 그 아쉬움이 해소되는 걸 느꼈다.

3. Finding : 어떤 인사이트를 얻었나#

Admin 페이지는 Django 프레임워크에서 기본으로 제공된다는 점을 알게 되었다.

4. Future action : 앞으로의 나#

앞으로도 계속 이 프레임워크를 가지고 놀면서 다양한 시도와 활용을 해보고, 고객이 원하는 기능과 내가 원하는 기능을 직접 구현할 수 있도록 이 스택을 내 것으로 만들어야겠다고 생각했다.
