Install 지옥#

2주차에는 간단한 이론 교육 후 Django 프레임워크를 설치하고 설정하는 시간을 가졌다. 별거 아닐 줄 알았는데 1시간 반 넘게 설치만 붙잡고 있었던 것 같다. Spring 프레임워크를 먼저 공부했던 나는 MVC 패턴처럼 Django만의 폴더 구조에도 어떤 새로운 패턴이 있을 거라 예상하고 있었는데, 제대로 된 설명 없이 갑자기 “웹페이지를 띄워보라"는 말을 들으니 정말 막막했다.

2주차 미션 이후..#

2주차 미션은 Django 프레임워크를 이용해 아기사자 관리 웹 페이지를 구현하는 것이었다.

🦁 아기사자 관리 웹 페이지 구현!!!!#

정기 세션 시간에 배운 내용을 바탕으로 직접 이것저것 만져보며 완성해봤다.

처음엔 Django의 폴더 구조와 URL 라우팅 방식이 낯설어서 헤맸다. 특히 html 템플릿 파일명 관련 오류를 계속 마주쳐서 애를 먹었는데, 폴더 구조나 파일명을 조금만 잘못 지정해도 바로 에러가 떠서 그 원인을 찾는 데만 한참을 썼던 것 같다. 그래도 직접 모델을 만들고 뷰와 템플릿을 연결해보면서 조금씩 감이 잡히기 시작했다. Spring에서 익숙했던 MVC 흐름과 완전히 다른 건 아니고, 결국 “요청을 받아서 → 처리하고 → 응답한다"는 웹 개발의 본질은 프레임워크가 달라도 비슷하다는 걸 몸으로 느낀 한 주였다.

이제는 이 프레임워크를 어떻게 활용해야 할지 조금씩 감이 잡힌다. 다음 단계를 위해 계속 달려가 보려 한다.
