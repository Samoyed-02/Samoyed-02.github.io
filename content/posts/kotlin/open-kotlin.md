## 시작하게 된 배경

취업 준비를 하다 보면 언젠가는 코딩테스트라는 벽을 마주하게 됩니다. 저에게는 그 벽이 조금 더 구체적이었어요. 목표로 하는 회사가 국내 핀테크/IT 기업들이 Kotlin 채택이 늘어나고 있다는 점이에요.

문제는 제가 Java도, Kotlin도 처음 다뤄보는 완전 초보였다는 점입니다. 어떤 언어로 알고리즘을 공부를 시작할 지 고민하다가, 다음과 같은 이유로 Kotlin을 첫 언어로 선택했습니다.

- 목표 기업들이 실제로 Kotlin을 프로덕션에서 사용하고 있다는 점
- 코딩테스트 준비에 자주 쓰이는 프로그래머스 플랫폼이 Kotlin을 잘 지원한다는 점
- 어차피 새로 배워야 한다면, 실무에서도 바로 써먹을 수 있는 언어를 택하는 게 효율적이라고 판단했다는 점

그래서 프로그래머스를 통해 문제를 풀고, IntelliJ IDEA에 로컬로 풀이 기록을 정리하는 방식을 공부를 시작했습니다.

## 공부하면서 체감한 Kotlin의 장점

아직  초보 단계지만, Java 문법과 비교되는 자료들을 찾아보고 직접 에러를 마주치며 부딪혀본 결과 몇 가지 장점을 체감할 수 있었습니다.

### 1. 간결한 문법

Java였다면 여러 줄이 필요했을 코드가 Kotlin에서는 훨씬 짧게 끝나는 경우가 많았습니다. 특히 알고리즘 문제 풀이처럼 짧은 시간 안에 로직을 구현해야 하는 상황에서는, 문법이 간결할 수록 실수할 여지도 줄어든다는 걸 느꼈어요.

### 2. Null Safey

val 과 var의 구분, 그리고 nullable 타입을 명시적으로 다뤄야 하는 구조 덕분에 예상치 못한 NullPointerException을 미연에 방지할 수 있다는 점이 인상 깊었습니다. 처음엔 타입 관련 에러 메세지가 낯설었지만, 오히려 컴파일 단계에서 문제를 잡아준다는 점이 안정감을 줬어요.

### 3. 풍부한 표준 라이브러리 함수

withIndex(), sorted(), digitToInt() 처럼 배열/문자열/컬렉션을 다루는 함수들이 잘 갖춰져 있어서, 알고리즘 문제에서 자주 등장하는 패턴(인덱스와 값을 함께 순회하기, 정렬하기, 문자를 숫자로 변환하기 등)을 짧게 표현할 수 있었습니다.

### 4. Java와의 상호 운용성

Kotlin은 JVM기반 언어라 Java 생태계의 자료와 라이브러리를 참고하기가 수월했습니다. Java 문법과 헷갈리는 지점(length() vs length, ArrayOf vs arrayOf 등) 이 오히려 두 언어의 차이를 명확히 이해하는 계기가 되기도 했고요.
